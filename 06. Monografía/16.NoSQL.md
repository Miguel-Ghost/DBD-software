# No-SQL

## Desarrollo conceptual

### Introducción

Redis, cuyo nombre es un acrónimo de **RE**mote **DI**ctionary **S**erver, es una base de datos en memoria que se destaca por su velocidad y versatilidad. A diferencia de las bases de datos tradicionales, Redis opera principalmente en memoria, lo que le permite realizar operaciones extremadamente rápidas. Se utiliza en una variedad de aplicaciones, desde caché y cola de mensajes hasta sistemas de recomendación y análisis en tiempo real.

### Características Claves de Redis

1. **Estructuras de Datos Avanzadas**:
   - **Cadenas (Strings)**: La estructura más básica, utilizada para almacenar texto o datos binarios.
   - **Listas (Lists)**: Secuencias de elementos ordenados que permiten operaciones como insertar, eliminar y acceder a elementos en los extremos.
   - **Conjuntos (Sets)**: Colecciones de elementos únicos que permiten operaciones de conjuntos como unión, intersección y diferencia.
   - **Mapas Hash (Hashes)**: Permiten almacenar objetos representados como pares clave-valor.
   - **Conjuntos Ordenados (Sorted Sets)**: Similar a los conjuntos, pero cada elemento está asociado a una puntuación que determina su orden.
   - **Bitmaps y HyperLogLogs**: Utilizados para operaciones bit a bit y estimaciones probabilísticas de cardinalidad, respectivamente.

2. **Persistencia**:
   - Redis ofrece varias opciones de persistencia para almacenar los datos en disco, lo que permite recuperar los datos tras un reinicio.
   - **Snapshots (RDB)**: Toman instantáneas de la base de datos en intervalos regulares.
   - **AOF (Append Only File)**: Registra cada operación de escritura, proporcionando un historial detallado de cambios que se puede reproducir para restaurar el estado de la base de datos.

3. **Alta Disponibilidad y Escalabilidad**:
   - **Replicación**: Redis soporta la replicación maestro-esclavo, lo que permite replicar datos entre múltiples nodos para aumentar la disponibilidad y distribuir la carga.
   - **Redis Sentinel**: Proporciona supervisión y conmutación por error automatizada para configurar clústeres de Redis altamente disponibles.
   - **Redis Cluster**: Permite dividir los datos entre múltiples nodos para escalar horizontalmente y manejar grandes volúmenes de datos.

4. **Atomicidad y Consistencia**:
   - Redis asegura la atomicidad de las operaciones sobre una sola clave.
   - **Transacciones**: Mediante comandos como MULTI, EXEC, y DISCARD, Redis permite agrupar múltiples operaciones en una sola transacción, asegurando que todas las operaciones se ejecuten de manera atómica.

5. **Baja Latencia**:
   - Redis está diseñado para ser extremadamente rápido, con tiempos de respuesta que suelen estar en el rango de microsegundos a milisegundos, lo que lo hace ideal para aplicaciones que requieren tiempos de respuesta rápidos.

### Casos de Uso de Redis

1. **Caché**:
   - Redis se utiliza comúnmente como caché para almacenar datos que se acceden con frecuencia, reduciendo la carga en las bases de datos primarias y mejorando el rendimiento de las aplicaciones.

2. **Cola de Mensajes**:
   - Gracias a sus estructuras de datos como listas y conjuntos ordenados, Redis se puede usar para implementar colas de mensajes y sistemas de procesamiento de tareas asíncronas.

3. **Gestión de Sesiones**:
   - Redis es ideal para almacenar sesiones de usuario en aplicaciones web, permitiendo un acceso rápido y eficiente a los datos de sesión.

4. **Ranking y Contadores**:
   - Las capacidades de Redis para manejar conjuntos ordenados y contadores lo hacen perfecto para implementar sistemas de ranking y métricas en tiempo real.

5. **Pub/Sub**:
   - Redis soporta un sistema de publicación y suscripción (Pub/Sub) que permite la comunicación entre diferentes partes de una aplicación mediante mensajes.

### Ventajas y Desventajas de Redis

**Ventajas:**
- **Velocidad**: Al operar en memoria, Redis ofrece un rendimiento excepcionalmente rápido.
- **Flexibilidad**: Soporte para múltiples estructuras de datos permite una amplia gama de aplicaciones.
- **Simplicidad**: La interfaz de Redis es sencilla y fácil de usar, lo que facilita su integración y uso.

**Desventajas:**
- **Limitaciones de Memoria**: Al ser una base de datos en memoria, la cantidad de datos que Redis puede manejar está limitada por la cantidad de memoria disponible en el sistema.
- **Persistencia Compleja**: La configuración y manejo de la persistencia pueden ser complejos, especialmente en escenarios de alta disponibilidad.

Proceso de Inserción:

1. Recopilación de Datos del Pedido: Cuando un cliente realiza un pedido, los detalles del pedido (como el código del pedido, código del representante, código del 2. empleado, tipo de pedido, estado del pedido, fecha de registro y código del ticket) se recopilan.

#### Modificación de Pedidos
La modificación de pedidos es crucial para actualizar el estado o cualquier otro detalle del pedido en función de las acciones del cliente o el procesamiento interno del negocio.

Proceso de Modificación:

1. Identificación del Pedido: Se identifica el pedido que necesita ser modificado, generalmente mediante su código único.
2. Modificación en Redis: Se utiliza el comando HSET para modificar el campo deseado del pedido.

``` redis
HSET pedido:1 cod_pedido_estado "C"
```
Beneficios:

1. Actualización en Tiempo Real: Los cambios en el estado del pedido se reflejan instantáneamente, mejorando la eficiencia del procesamiento de pedidos.
2. Facilidad de Implementación: Las actualizaciones se pueden realizar sin afectar otros campos del pedido.
